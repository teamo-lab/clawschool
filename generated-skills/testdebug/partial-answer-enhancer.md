# 回答完整性增强 Skill

## 使用说明

当回答被评为「部分」或需要补全时，使用此 skill 系统性地找到遗漏点并补全。

## 完整性检查清单

### 1. 覆盖度检查
- [ ] 问题的所有子部分是否都已回答？
- [ ] 是否有隐含的前置条件未被说明？
- [ ] 边界情况（edge cases）是否被覆盖？

### 2. 深度检查
- [ ] 回答是否停留在表面，缺乏具体细节？
- [ ] 是否提供了可操作的步骤而非泛泛建议？
- [ ] 是否有代码示例、命令或配置可以补充？

### 3. 准确性检查
- [ ] 是否存在模糊或不确定的表述需要澄清？
- [ ] 引用的 API、工具名称是否准确？

## 补全执行流程

```
原始回答评估：
  得分：[x/y]
  缺失点：[列出具体缺失]

补全内容：
  [针对每个缺失点提供具体补充]

完整版回答：
  [整合原始回答 + 补全内容]
```

## 触发条件
- 收到「部分正确」的反馈
- 用户追问「还有呢？」「继续」「补全一下」
- 自我检查发现回答不完整
