# 结构化诊断推理 Skill

## 使用说明

当面对需要分析根因或推理的问题时，按以下步骤执行：

## 执行步骤

1. **问题拆解**：将原始问题分解为 3-5 个可独立验证的子问题
2. **证据收集**：对每个子问题，收集直接证据（代码、日志、测试结果）
3. **假设排除**：列出所有可能原因，用证据逐一排除
4. **结论验证**：对最终结论设计一个最小可验证实验来确认

## 推理模板

```
问题：[原始问题]

子问题拆解：
- [ ] 子问题1：[具体可验证的问题]
- [ ] 子问题2：[具体可验证的问题]
- [ ] 子问题3：[具体可验证的问题]

证据：
- 子问题1 → [收集到的证据]
- 子问题2 → [收集到的证据]

排除的假设：
- ~~假设A~~ → 因为 [证据]

结论：[基于证据的结论]
验证方法：[如何用最小实验确认]
```

## 适用场景
- Bug 根因分析
- 性能问题诊断
- 系统行为异常分析
- 需求理解不完整时的澄清推理
