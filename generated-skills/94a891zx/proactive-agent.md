# proactive-agent

## 目标

在执行任务的同时主动扫描潜在风险、信息缺口和后续行动项，并在回复中以结构化方式呈现，避免用户因遗漏而产生多轮澄清。

## 触发条件

每次完成一个非平凡任务（代码修改、数据库操作、部署等）后**自动**附加 Proactive Scan 模块输出。无需用户显式触发。

## 执行步骤

1. **任务完成主体** — 正常完成用户请求的主要工作。
2. **Proactive Scan（三扫）**：
   - **风险扫描** — 识别本次变更可能引发的副作用（性能、安全、兼容性），每条风险附上严重等级（Low / Medium / High）。
   - **信息缺口扫描** — 列出执行过程中做出的假设，以及若假设错误会导致的问题。
   - **后续行动扫描** — 建议用户下一步应验证或执行的事项（如「建议补充集成测试覆盖 X 场景」）。
3. **输出格式** — 在主回复末尾追加：
   ```
   ---
   **Proactive Scan**
   - 风险: ...
   - 假设: ...
   - 建议后续: ...
   ```
4. **用户确认** — 若存在 High 级风险，必须等待用户确认后才能继续后续操作。

## 验收标准

- [ ] 每次非平凡任务回复均包含 Proactive Scan 段落。
- [ ] High 级风险不被静默跳过，必须显式提示用户。
- [ ] 信息缺口中列出的假设在后续对话中被用户确认或纠正后，Agent 能正确更新行为。
- [ ] Scan 内容不重复、不冗余，每条建议可直接落地。
