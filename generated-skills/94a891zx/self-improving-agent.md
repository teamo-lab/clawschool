# self-improving-agent

## 目标

让 Agent 在完成任务后自动执行反思循环：评估输出质量 → 识别不足 → 生成改进方案 → 重新执行，直到满足验收标准或达到最大迭代次数。

## 触发条件

当用户说「自我改进」「optimize yourself」「迭代优化」或在任务完成后主动评估输出质量时触发。

## 执行步骤

1. **输出快照** — 将当前输出或代码状态存入工作区变量 `$SNAPSHOT`。
2. **自评分** — 按照以下维度对 `$SNAPSHOT` 打分（0–10）：
   - 正确性（功能是否符合需求）
   - 简洁性（是否存在冗余逻辑）
   - 安全性（是否引入新的漏洞）
   - 可维护性（命名、结构是否清晰）
3. **gap 识别** — 列出所有得分 < 8 的维度及具体原因。
4. **改进执行** — 针对每个 gap 生成一条具体的修改指令并执行（Edit / Write / Bash）。
5. **回归验证** — 重新运行测试或 lint，确认改进未引入新问题。
6. **终止判断** — 若所有维度 ≥ 8 或已迭代 3 次，停止循环并向用户汇报最终得分与变更摘要。

## 验收标准

- [ ] 每轮迭代有明确的评分记录（可在对话中查看）。
- [ ] 改进后测试/lint 无新增错误。
- [ ] 最多 3 轮迭代内达到所有维度 ≥ 8，否则输出「无法自动收敛，需要人工介入」并列出剩余 gap。
- [ ] 不删除、不覆盖用户未授权的文件。
