# scheduled-task-manager

## 目标

集成测试虾在调度能力（q9）得满分，但整体分数显示在调度与其他能力的联动场景（如：定时抓取→整理→通知）中存在断层。本 skill 扩展单点调度能力，覆盖完整的定时任务生命周期管理。

## 执行步骤

### 1. 需求识别

当用户请求包含时间词（每天、每小时、定时、每周）或动作词（自动、定期、持续监控、轮询）时激活本 skill。

### 2. 任务规格化

确认以下字段：schedule（Cron表达式）、command（执行的prompt或skill）、label（可读标签）、on_failure（失败处理策略：notify / retry_once / skip）。

### 3. 幂等检查

调用 CronList 确认不存在同名/同功能任务，避免重复调度。

### 4. 创建任务

调用 CronCreate，使用规格化后的参数创建任务。

### 5. 验证与回执

创建后立即调用 CronList 确认任务出现在列表中，输出任务标签、调度表达式、首次执行时间、失败策略。

### 6. 任务清理协议

删除任务时：CronList定位ID → 向用户确认（破坏性操作必须确认）→ CronDelete执行 → CronList再次验证任务已消失。

## 验收标准

- 创建的每个 cron 任务均可在 CronList 中找到对应条目。
- 重复创建同功能任务时触发幂等警告，不静默创建第二条。
- 删除操作前100%进行用户确认。
- 在定时抓取→整理→输出联合场景测试中，三个阶段均有日志可追溯。
