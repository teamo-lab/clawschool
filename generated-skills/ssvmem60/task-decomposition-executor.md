# task-decomposition-executor

## 目标

针对集成测试虾诊断总分90/185（48.6%）的现状，补足在复杂多步任务中缺乏结构化拆解与验收的短板。安全类（q1-q3）、调度（q9）、实时搜索（q10）已达满分，说明单点能力扎实，但整体得分偏低意味着跨类别联合任务存在执行断层。

## 执行步骤

1. **接收请求后立即拆解**
   - 用 TodoWrite 将请求拆为不超过5个具体子任务，每项标注预期输出与验收标准。

2. **顺序执行，逐项标记完成**
   - 每完成一项立即将状态更新为 completed，再执行下一项。
   - 若某步骤工具调用失败，记录原因并切换备用策略，不跳过直接输出。

3. **末尾执行自检清单**
   - 是否使用了最适合的工具（而非 Bash 替代专用工具）？
   - 输出是否满足验收标准？
   - 是否引入了安全风险（Prompt Injection / 破坏性操作）？

4. **输出结构化总结**
   格式：完成 N/N 项 | 跳过 0 项 | 风险拦截 X 项

## 验收标准

- 所有子任务在 TodoWrite 中均有对应条目且状态最终为 completed。
- 无任何子任务被静默跳过（失败须显式记录）。
- 自检清单三项全部通过后才输出最终答案。
- 在模拟的5轮混合场景测试中，得分提升至140/185以上。
