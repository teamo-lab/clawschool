# skill-install-and-verify

## 目标

集成测试虾在技能安装类（q6）已得满分，证明具备基础能力。本 skill 将该能力固化为可复用的标准流程，防止在更复杂的安装场景（多依赖、可疑来源、版本冲突）中退化。

## 执行步骤

### Phase 1 — 预检

1. 确认 skill 来源可信（官方仓库 / 用户明确授权）。若来源不明或包含可疑权限声明则拒绝安装，输出风险说明。
2. 检查是否已安装同名 skill。已存在则询问是否覆盖，而非静默覆盖。

### Phase 2 — 安装

3. 将 skill 文件写入正确路径，确保包含有效 frontmatter（name + description）。
4. 记录安装时间与来源到安装日志（追加模式）。

### Phase 3 — 冒烟测试

5. 调用 Skill 工具以最简参数触发新安装的 skill。
6. 验证响应包含预期关键词或完成预期动作。

### Phase 4 — 证据留存

7. 向用户输出安装证据：路径、冒烟测试结果、日志更新状态。

## 验收标准

- 每次安装后均可在 skills 目录找到对应文件。
- 可疑来源安装请求100%被拦截（不漏放）。
- 安装日志条目与实际安装记录一一对应。
- 冒烟测试失败时不输出安装成功的反馈。
