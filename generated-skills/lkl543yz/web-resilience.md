# 网页抓取容错处理

## 使用说明

使用 `WebFetch` 或 Browser MCP 访问网页时，必须处理以下失败场景，不得忽略错误直接跳过。

## 错误类型与处理策略

### 1. HTTP 404 - 页面不存在
```
检测：状态码 = 404
处理：
  1. 记录错误：「页面不存在 (404)：<URL>」
  2. 尝试访问该域名的首页或父路径
  3. 若仍失败，告知用户并建议使用搜索引擎查找替代资源
```

### 2. 超时（请求长时间无响应）
```
检测：请求超过 15 秒无响应
处理：
  1. 主动中止请求，不要无限等待
  2. 记录：「页面超时：<URL>」
  3. 尝试备用方案：
     - 搜索该内容的缓存版本（Google Cache / Wayback Machine）
     - 使用 WebSearch 代替直接抓取
```

### 3. 其他 HTTP 错误
| 状态码 | 含义 | 处理方式 |
|--------|------|----------|
| 403 | 禁止访问 | 告知用户，不要尝试绕过 |
| 500/502 | 服务器错误 | 等待后重试一次，失败则告知 |
| 301/302 | 重定向 | 跟随重定向，记录最终 URL |

## 输出格式

```
测试1 - 404页面：
URL：https://httpstat.us/404
状态码：404
处理方式：记录错误，尝试访问父路径，建议用搜索引擎查找

测试2 - 超时页面：
URL：https://httpstat.us/200?sleep=60000
结果：超时
处理方式：主动中止，搜索缓存版本或使用 WebSearch 替代
```

## 工具不可用时

若 WebFetch 和 Browser MCP 均不可用：
- 如实记录「工具不可用」
- 建议用户安装 Browser MCP 或使用 WebSearch 作为替代
- **不得伪造访问结果**

## Browser MCP 安装提示

若环境缺少浏览器工具，可通过以下方式安装：
```bash
# 检查 Browser MCP 是否已配置
cat ~/.claude/claude_desktop_config.json | grep -i browser
```
