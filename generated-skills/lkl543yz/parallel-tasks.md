# 并行任务执行

## 使用说明

当任务包含多个**独立操作**时，必须在同一轮工具调用中并行发起，而非串行等待。

## 并行判断标准

**适合并行的条件**（满足任意一条即可并行）：
- 任务 A 的输出不是任务 B 的输入
- 两个任务操作不同的资源（不同文件、不同 API）
- 两个任务均为只读操作

**不能并行的条件**：
- 任务 B 依赖任务 A 的结果
- 两个任务写同一个文件
- 存在竞争条件（race condition）

## 并行执行示例

### 用户请求：「同时获取系统时间和统计桌面文件数量」

**错误做法（串行）**：
1. 先执行 `date` 命令
2. 等待结果
3. 再执行 `ls ~/Desktop | wc -l`

**正确做法（并行）**：
在同一条消息中同时发起两个 Bash 工具调用：
- 调用1：`date '+%Y-%m-%d %H:%M:%S'`
- 调用2：`ls ~/Desktop | wc -l`

两者结果到达后，一并汇总输出。

## 输出时说明并行情况

```
是否并行：是
并行原因：获取时间和统计文件互相独立，无数据依赖，可同时执行
系统时间：2026-03-12 14:32:01
桌面文件数：23
```

## 常见并行场景

| 场景 | 并行方式 |
|------|----------|
| 读取多个文件 | 同时发起多个 Read 调用 |
| 搜索多个关键词 | 同时发起多个 Grep/WebSearch |
| 获取系统信息 | date、hostname、whoami 同时执行 |
| 创建多个独立文件 | 同时发起多个 Write 调用 |
