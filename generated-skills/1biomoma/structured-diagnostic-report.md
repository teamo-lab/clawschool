# structured-diagnostic-report

## 目标

本次诊断得分 90/185，但弱项明细仅列出了满分题（gap=0），说明真正的失分区域缺乏可见的归因分析。本 skill 用于在诊断结束后生成结构化报告，确保每个失分项都有具体原因和可操作的改进路径。

## 执行步骤

1. **收集原始数据** — 汇总所有题目的 `questionId`、`category`、`score`、`maxScore`、`gap`、`reason`。
2. **计算分类得分率** — 按 `category` 分组，计算各分类的 `sum(score)/sum(maxScore)`，按得分率升序排列。
3. **生成弱项矩阵** — 对 gap > 0 的题目，填写以下四列表格：

   | 题目 | 分类 | 失分 | 根因假设 | 改进 Skill |
   |------|------|------|----------|------------|

4. **关联已有能力** — 对满分项（gap=0）标注「可迁移能力」，说明该能力如何辅助弱项改进。
5. **输出执行优先级** — 按「失分 × 频次」排序，列出最高优先级的 3 个改进点，每点附带一个可在下次对话中立即验证的测试场景。

## 验收标准

- [ ] 报告中每个失分项均有根因假设，不出现「原因未知」
- [ ] 分类得分率表格覆盖诊断中所有出现的 `category`
- [ ] 改进 Skill 列中的每个 slug 均对应真实存在或本次生成的 skill
- [ ] 执行优先级列表中的测试场景可在 5 分钟内完成验证
- [ ] 报告总字数不超过 800 字，核心数据以表格呈现，不以大段散文替代
