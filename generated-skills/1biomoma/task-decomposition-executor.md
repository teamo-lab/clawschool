# task-decomposition-executor

## 目标

诊断显示总分 90/185（≈48.6%），已确认的满分项均属于单轮判断类题目（安全识别、工具调用、即时搜索）。低分区集中在需要跨步骤推理、状态追踪和结果验证的任务上。本 skill 通过强制结构化拆解流程，提升复杂任务完成率。

## 执行步骤

1. **接收任务** — 将用户请求拆解为不超过 5 个原子子任务，每个子任务必须有明确的完成判定条件。
2. **依赖排序** — 标注子任务间的依赖关系，独立任务并行执行，有依赖的串行执行。
3. **逐步执行** — 每完成一个子任务立即用 `TodoWrite` 标记 `completed`，不批量标记。
4. **检查点验证** — 在每个子任务完成后，用一句话陈述「实际产出 vs 预期产出」，若不符则就地修正，不跳过。
5. **最终汇总** — 所有子任务完成后，输出一个三列表格：子任务 | 状态 | 关键证据。

## 验收标准

- [ ] 每个子任务的完成判定条件在执行前已明确写出
- [ ] 没有出现「批量完成」或跳过检查点的情况
- [ ] 最终汇总表格中每行均有可验证的证据（文件路径、命令输出片段、URL 等）
- [ ] 若任意子任务失败，整体状态标记为 `PARTIAL`，并列出阻塞原因
