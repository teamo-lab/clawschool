# realtime-search-with-date-anchor

## 目标

诊断显示「当日新闻整理」满分，说明已具备实时搜索能力，但为了将该能力稳定复现并防止在高压任务下退化为训练数据幻觉，需要将日期锚定流程固化为标准操作。

## 适用场景

- 用户询问「今天」「最近」「当前」「最新」等时间敏感信息
- 需要汇总新闻、行情、版本更新、政策变化等动态内容
- 任何答案可能因日期不同而显著变化的问题

## 执行步骤

1. **日期声明** — 在搜索前，明确输出当前日期（从系统上下文读取，格式：`YYYY-MM-DD`），不依赖模型内部估算。
2. **含日期搜索** — `WebSearch` 查询中显式包含日期限定词，例如 `"2026-03" site:reuters.com`，避免返回旧结果。
3. **多源交叉验证** — 对重要事实至少搜索 2 个独立来源；若来源日期不一致，以较新的可信来源为准并注明差异。
4. **标注信息时效** — 输出内容中，每条关键信息后注明来源日期，格式：`（来源日期：YYYY-MM-DD）`。
5. **不确定时声明** — 若搜索结果不足以确认时效性，主动告知用户「当前搜索结果覆盖截至 [日期]，建议自行核实最新状态」。

## 验收标准

- [ ] 时间敏感回答中，没有使用无来源日期的绝对性陈述
- [ ] 每次 `WebSearch` 调用的 query 包含明确的时间限定
- [ ] 输出的新闻/资讯条目均附有来源日期
- [ ] 当搜索结果与系统日期相差超过 7 天时，有主动提示
