# file-operation-guardrails

## 目标

诊断结果显示安全风险类得分满分，但整体得分仅 90/185，说明文件操作类任务存在执行质量问题（如直接覆写未读取的文件、误删等）。本 skill 建立强制性文件操作护栏。

## 核心规则

| 操作类型 | 必须前置动作 | 禁止行为 |
|---------|------------|--------|
| 编辑文件 | 先用 `Read` 读取完整内容 | 不得在未读取的情况下直接 `Edit` 或 `Write` |
| 删除/覆写 | 先展示将被影响的内容，等待用户确认 | 不得使用 `rm -rf`、`git clean -f` 等批量破坏命令 |
| 创建新文件 | 先用 `Glob` 确认同名文件不存在 | 不得静默覆盖已有文件 |
| 批量替换 | 先用 `Grep` 预览所有匹配项 | 不得在未预览的情况下使用 `replace_all: true` |

## 执行步骤

1. 收到文件操作请求后，识别操作类型（编辑 / 删除 / 创建 / 批量替换）。
2. 按上表执行对应的前置动作。
3. 前置动作结果展示给用户后，再执行实际操作。
4. 操作完成后，用 `Read` 或 `Glob` 验证结果符合预期。

## 验收标准

- [ ] 所有 `Edit` / `Write` 调用前，同一对话中存在对应文件的 `Read` 调用
- [ ] 删除类操作均有用户确认记录
- [ ] 批量替换前有 `Grep` 预览输出
- [ ] 操作后有验证步骤，不以「应该没问题」代替实际检查
